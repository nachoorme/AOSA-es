%Many claims are made about how certain tools, technologies, and
%practices improve software development. But which are true, and which
%are merely wishful thinking? In \emph{Making Software}, leading
%researchers and practitioners present chapter-length summaries of key
%empirical findings in software engineering, and answer questions like:
Se hacen muchas afirmaciones acerca de cómo ciertas herramientas, tecnologías
y prácticas mejoran el desarrollo del software. Pero cuáles son ciertas
y cuáles son sólo buenos deseos? En \emph{Making Software}, investigadores y
profesionales líderes presentan resúmenes por capítulo acerca de importantes
hallazgos empíricos en ingeniería de software y responden a preguntas tales como:

\vspace{-0.1cm}

\begin{aosaitemize}

%\item Are some programmers really ten times more productive than others?
\item ¿Realmente algunos programadores son diez veces más productivos que otros?

%\item Does writing tests first help you develop better code faster?
\item ¿Escribir tests al principio ayuda a desarrollar mejor código y más rápido?

%\item Can code metrics predict the number of bugs in a piece of software?
\item ¿Pueden las métricas de código predecir la cantidad de errores en un componente de software?

%\item Does using design patterns actually make software better?
\item ¿Utilizar patrones de diseño realmente hace mejor al software?

%\item What effect does personality have on pair programming?
\item ¿Cómo afecta la personalidad en la programación en parejas (pair-programming)?

%\item What matters more: how far apart people are geographically, or how far apart they are in the org chart?
\item ¿Qué es más importante: qué tan lejos están las personas geográficamente, o qué tan lejos están en
el cuadro organizacional?

\end{aosaitemize}
